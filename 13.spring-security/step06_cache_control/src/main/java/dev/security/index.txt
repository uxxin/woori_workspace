스프링 시큐리티가 제공해주는 CacheControl 옵션에 따른 차이 비교

1. SecurityConfig에서 cacheControl() 옵션 없이 실행
2. 인증된 사용자는 자신의 (민감한?) 리소스인 "hello"를 조회하기 위해 인증 수행
3. 인증이 성공할 경우 사용자는 "hello"를 조회할 수 있음
4. 내용을 확인한 사용자는 로그아웃을 마치고 사라짐
5. 악의적인 사용자가 뒤로가기 버튼을 눌러도 캐싱이 되지 않았기 때문에 내용 확인 불가능

Cache Control 옵션 비활성화 후 테스트 시나리오

1. SecurityConfig에서 cacheControl() 옵션을 비활성화하여 스프링 시큐리티의 캐싱 비활성화 옵션을 비활성화
2. 인증된 사용자는 자신의 (민감한?) 리소스인 "hello"를 조회하기 위해 인증 수행
3. 인증이 성공할 경우 사용자는 "hello"를 조회할 수 있음
4. 내용을 확인한 사용자는 로그아웃을 마치고 사라짐
5. 악의적인 사용자가 뒤로가기 버튼을 눌렀을 경우, 캐싱이 되어있기 때문에 다른 사용자의 정보를 조회 가능
